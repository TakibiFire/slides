# Context

This directory contains many Slidev projects in sub-directories.
See ./deploy.sh about how to compile each project.

The output directory is ~/projects/takibi-fire/public_root/slides/"$project"/.
Then we use FireBase to make a static file hosting service.
You are allowed to see /Users/jumpei/projects/takibi-fire/firebase.json about its configuration.

For example, firebase.json contains the following 

```
        {
          "source": "/slides/fire-4-stages/**",
          "destination": "/slides/fire-4-stages/index.html"
        },
```

This allows

`https://takibi-fire.com/slides/fire-4-stages`, `https://takibi-fire.com/slides/fire-4-stages/`, `https://takibi-fire.com/slides/fire-4-stages/1`, `https://takibi-fire.com/slides/fire-4-stages/2`, etc. all load fire-4-stages/index.html. Slidev SPA then rewrites the URL to `https://takibi-fire.com/slides/fire-4-stages/1` to show the first slide page. Right now, the canonical URL is not set in the index.html, which might be some clue for the problmes below.

sitemap.xml correctly contains `https://takibi-fire.com/slides/fire-4-stages/`. Note that this is auto-generated by `src/sitemap-generator.js`.
